DEMO FLOW - Insurance Intelligence Agent

Duración total estimada: 12-15 minutos

1. Semantic View (2 min)

Mostrar: Snowsight → CAVIDA_POC.ANALYTICS → Semantic Views → INSURANCE_INTELLIGENCE
Acción: Abrir la definición YAML y mostrar las 5 tablas, métricas, y sinónimos en portugués
Punto: "Una sola fuente de verdad semántica que alimenta al agente, Power BI, y cualquier app. Databricks no tiene esto."

2. Cortex Agent en Acción (4 min)

Mostrar: Streamlit App (insurance_agent_app.py)
Preguntas sugeridas:
  1. "Qual é a prima total do portfolio?" → muestra total premium
  2. "Quantas apólices estão ativas por status?" → breakdown por status
  3. "Qual é a evolução mensal de novas apólices?" → time series
  4. "Quantas apólices foram canceladas?" → terminated analysis
Punto: "El usuario pregunta en lenguaje natural (portugués o español) y el agente genera SQL optimizado sobre 24M registros. 99% accuracy vs 88% de Databricks Genie."

3. Governance - PII & Masking (3 min)

Mostrar: Snowsight SQL Worksheet
Queries:
  1. SELECT * FROM TABLE(INFORMATION_SCHEMA.TAG_REFERENCES_ALL_COLUMNS('CAVIDA_POC.POC_LOOK.EMPLOYEE', 'TABLE')) WHERE TAG_NAME = 'PII_TYPE' → muestra 3 columnas clasificadas
  2. USE ROLE PUBLIC; SELECT EMPLOYEE_ID, X_CENTRAL_TAX_NO FROM CAVIDA_POC.POC_LOOK.EMPLOYEE LIMIT 5; → muestra ***MASKED***
  3. USE ROLE ACCOUNTADMIN; SELECT EMPLOYEE_ID, X_CENTRAL_TAX_NO FROM CAVIDA_POC.POC_LOOK.EMPLOYEE LIMIT 5; → muestra datos reales
Punto: "Un tag = una política de masking para todas las columnas sensibles. Con Agent Identity (GA), el propio agente tiene su identidad y se le aplican las políticas. Databricks no tiene ni AI Guardrails ni Agent Identity."

4. Task DAG con Alertas AI (3 min)

Mostrar: Snowsight → CAVIDA_POC.ORCHESTRATION → Tasks
Acciones:
  1. Mostrar el DAG visual (DQ_CHECK_ROOT → 2 children)
  2. SELECT * FROM CAVIDA_POC.ORCHESTRATION.PIPELINE_MONITOR_LOG; → resultados con AI_SUMMARY
  3. Mostrar la columna AI_SUMMARY generada por Cortex LLM
  4. Mostrar la Alert con notificación email
Punto: "Snowflake usa Cortex LLM dentro del propio Task para generar explicaciones inteligentes de cada alerta. En Databricks esto requiere un notebook separado + MLflow + infraestructura adicional."
